# Contributing to AdvisoryHub

- The **[contributor guide](docs/contributing/README.md)** covers the local
  dev environment, tests, the code-quality gates, and the commit/PR
  conventions.
- The **[release runbook](docs/contributing/releasing.md)** is the maintainer
  guide for cutting a release.

Before making non-trivial changes, read the relevant file in
[`docs/specification/`](docs/specification/) — the single source of truth for
what this system *is* and *does* — and cite `INV-*` IDs in commits and PRs.
