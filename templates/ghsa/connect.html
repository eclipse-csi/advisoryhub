GitHub App configuration

AdvisoryHub talks to GitHub as a registered GitHub App with repository_security_advisories: read & write and metadata: read. The App is installed per-repo by Eclipse org admins. The private key (loaded from the deployment environment) is the only sensitive credential and is never persisted to the DB.

How to install

  1. Register the AdvisoryHub GitHub App on the Eclipse organisation.
  2. Grant the two permissions above; deny everything else.
  3. Set the webhook URL to https://<your-host>/ghsa/webhook/ and copy the webhook secret into GITHUB_APP_WEBHOOK_SECRET.
  4. Subscribe the App to at least the installation, installation_repositories, and repository_advisory events.
  5. Install the App on the specific repositories that should be visible to AdvisoryHub. PMI's project↔repo mapping decides which repos a given AdvisoryHub project will discover GHSAs from.
  6. Copy the App id into GITHUB_APP_ID. Mount the App private key as a file and point GITHUB_APP_PRIVATE_KEY_PATH at it.
  7. Flip GHSA_FEATURE_ENABLED=True, then click Rescan installations above (or wait for the first installation.created webhook).
